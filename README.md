# 12306 余票查询 · 买长乘短

网页版余票查询：输入出发地 / 目的地、勾选车次类型和座位类型并查询。
当目的区段 **B→C 直达没票** 时，自动尝试同一车次的：

- **A→C**：提前一站（或多站）上车
- **B→D**：延后一站（或多站）下车
- **A→D**：两头都延伸

有结果就在页面上列出对应的延伸方案与余票数量（即「买长乘短」）。

## 运行

```bash
# 首次：创建虚拟环境并安装依赖
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

# 启动
.venv/bin/python app.py
```

打开浏览器访问 http://127.0.0.1:5000

## 文件说明

| 文件                  | 作用                                                       |
|-----------------------|------------------------------------------------------------|
| `app.py`              | Flask 后端，提供 `/`、`/api/stations`、`/api/query` 接口    |
| `ticket.py`           | 核心逻辑：余票查询、经停站查询、买长乘短延伸聚合            |
| `templates/index.html`| 查询页面（表单 + 结果展示）                                |
| `monitor.py`          | 原有的命令行余票监控脚本（独立，未改动）                   |

## 说明 / 注意

- 数据来自 12306 官网公开查票接口，仅供个人查询参考。
- 首次查询会先访问 12306 init 页面获取 Cookie，否则接口会返回错误页。
- 含延伸的查询会发起多次请求，已内置轻微限速（每次间隔 0.25s）和请求上限，
  请避免频繁刷新，以免被 12306 风控限速。
- 「买长乘短」只在你的区段处于车次**中间段**时才可能有方案；若你的出发站是该车
  始发站、到达站是终到站，则两头无法再延伸（属正常情况）。
- 部分区段 12306 不单独售卖（停靠 ≠ 该区段可购票），这类延伸会自动跳过。
