# 12306 余票查询 · 买长乘短

网页版余票查询：输入出发地 / 目的地、勾选车次类型和座位类型并查询。
当目的区段 **B→C 直达没票** 时，自动尝试同一车次的：

- **A→C**：提前一站（或多站）上车
- **B→D**：延后一站（或多站）下车
- **A→D**：两头都延伸

有结果就在页面上列出对应的延伸方案与余票数量（即「买长乘短」）。


## 主要功能

- **余票查询 + 买长乘短**：直达没票时自动找「提前上车 / 延后下车」的可购区段。
- **多日期查询**：填结束日期可一次查多天（最多 5 天）。
- **票价显示与筛选**：每个有票座位显示票价，可设价格上限、按价格/出发到达时间/耗时/余票排序。
  （票价接口对非登录会话限流较严，按需/渐进加载，偶有缺失可点「¥查价」补拉。）
- **浏览器端监控**：开着网页定时重查，有票弹浏览器通知 + 提示音，可自动弹开下单页。
- **☁️ 服务端托管监控**：监控任务跑在后端线程，**关掉网页也继续跑**，有票直接推微信；
  进程重启自动恢复 running 任务。需先配置微信推送。
- **微信通知**：PushPlus / Server酱 / 企业微信群机器人，推送与网页一致的「结果卡片」。
- **去 12306 下单 / 候补**：每趟车生成官方下单页深链，已登录则直接进登录态。

## 运行

```bash
# 首次：创建虚拟环境并安装依赖
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

# 启动
.venv/bin/python app.py
```

打开浏览器访问 http://127.0.0.1:5001 （可用环境变量 `PORT` 改端口）

### ⚠️ 服务默认启用访问令牌

本服务带自动登录态与自动下单能力，**任何能访问到它的人都能用你的 12306 账号下单**。

服务会读取或生成 `.app_token`（仅本机可读、已 gitignore）作为访问令牌；`start.sh` / `share.sh` 会在启动时打印出来。浏览器打开页面后点右上角「🔑 令牌」填入同一串即可（之后浏览器记住）。

```bash
./start.sh     # 本机使用：会打印令牌
./share.sh     # 对外分享：连同公网网址一起打印令牌，发给对方
```

- 仅本机使用：也建议填写令牌；`start.sh` 和手动启动都会打印当前令牌。
- 手动运行 `.venv/bin/python app.py`：也会自动读取或生成 `.app_token` 并在终端打印。
- 经 cloudflared / 端口转发对外暴露时：**必须**带令牌（`share.sh` 已强制设置），否则任何拿到公网地址的人都可能操作你的账号。

手动指定令牌（不走脚本时）：
```bash
APP_TOKEN='换成你自己的强随机串' .venv/bin/python app.py
```

其它环境变量：`HOST`（监听地址，默认 `127.0.0.1`）、`PORT`（端口）、`FLASK_DEBUG`（对外分享时务必不要开）、`ORDER_MAX_PARALLEL_TICKS`（自动抢票任务同时查询轮数，默认 `2`）。
敏感文件均已 gitignore，并会用 `0600` 权限落盘。安装了 `cryptography` 时，`login_session.json` 会整体加密，`order_jobs.json` / `monitor_jobs.json` 中的推送 token 会字段级加密。

## 文件说明

| 文件                  | 作用                                                       |
|-----------------------|------------------------------------------------------------|
| `app.py`              | Flask 后端：查询 / 站点 / 票价 / 下单链接 / 推送 / 监控接口 |
| `ticket.py`           | 核心逻辑：余票、经停站、票价、买长乘短延伸、官方深链        |
| `monitor_service.py`  | 服务端常驻监控：后台线程定时查 + 有票推微信 + 持久化恢复    |
| `order_service.py`    | 自动抢票任务管理：轮询余票、触发占座、持久化任务状态         |
| `cryptobox.py`        | 本地敏感字段加密，密钥保存在 gitignore 的 `.secret.key`      |
| `persist.py`          | JSON 防抖落盘与 `0600` 权限原子写                            |
| `notify.py`           | 微信推送（PushPlus / Server酱 / 企业微信）与卡片渲染        |
| `templates/index.html`| 查询页面（表单 + 结果 + 托管监控面板）                     |
| `monitor.py`          | 原有的命令行余票监控脚本（独立，未改动）                   |
| `monitor_jobs.json`   | 运行时生成：托管任务配置 + 状态（含 token，已 gitignore）  |

## 服务端托管监控用法

1. 在页面填好出发/目的/日期、座位等条件，并在「微信通知」里配好渠道 + token（先点「测试」确认能收到）。
2. 点「☁️ 服务端托管监控」里的 **＋ 用当前条件创建托管任务**。
3. 之后可**关闭网页**，只要运行 `app.py` 的电脑别关机，后端就持续查、有票推微信。
4. 面板里可查看每个任务的轮次/上次检查/累计命中，并「查看最近结果 / 停止 / 删除」。

## 说明 / 注意

- 数据来自 12306 官网公开查票接口，仅供个人查询参考。
- 首次查询会先访问 12306 init 页面获取 Cookie，否则接口会返回错误页。
- 含延伸 / 票价的查询会发起多次请求，已内置限速闸门和请求上限，请避免频繁刷新以免被风控限速。
- 票价接口（queryTicketPrice）对非登录会话限流较严，无法一次拉全；票价会按需渐进加载、服务端缓存，缺失项可点「¥查价」重试。
- 托管任务的 token 仅保存在本地 `monitor_jobs.json`；任务越多每轮越慢，**监控间隔建议 ≥ 30 秒**。
- 「买长乘短」只在你的区段处于车次**中间段**时才可能有方案；若出发站是始发站、到达站是终到站，则两头无法再延伸（属正常情况）。
- 部分区段 12306 不单独售卖（停靠 ≠ 该区段可购票），这类延伸会自动跳过。
